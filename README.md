# Virax Store Bot (@Viraxstorebot)

بوت تيليجرام كامل لمتجر يعمل بنظام نقاط، مبني بمكتبة `python-telegram-bot`.

## 1. هيكل المشروع

```
viraxstorebot/
├── main.py              # نقطة التشغيل الرئيسية
├── config.py            # كل الإعدادات (التوكن، الأدمن، إلخ)
├── database.py          # طبقة SQLite (المستخدمين، المنتجات، المشتريات)
├── keyboards.py         # لوحات المفاتيح (القائمة، أزرار المتجر...)
├── requirements.txt     # المكتبات المطلوبة
└── handlers/
    ├── user.py          # منطق المستخدم العادي
    ├── admin.py         # منطق لوحة تحكم الأدمن
    └── dev_secret.py    # مفتاح تجربة المطور
```

## 2. التشغيل على الهاتف (Pydroid 3)

1. افتح تطبيق **Pydroid 3**.
2. من القائمة الجانبية اذهب إلى **Pip** وثبّت المكتبة التالية:
   ```
   python-telegram-bot
   ```
   (تأكد من اختيار أحدث إصدار — الإصدار 21 أو أعلى).
3. انسخ كل مجلد `viraxstorebot` إلى داخل ذاكرة الهاتف (مثلاً داخل مجلد `Pydroid3`).
4. افتح ملف `main.py` من داخل Pydroid 3.
5. اضغط على زر التشغيل ▶️ أسفل الشاشة.
6. إذا ظهرت رسالة `🚀 البوت يعمل الآن...` في الطرفية (Console)، فالبوت يعمل بنجاح.
7. اترك تطبيق Pydroid 3 مفتوحاً (أو في الخلفية) طالما تريد إبقاء البوت شغّالاً.

> ملاحظة: قاعدة البيانات `store.db` يتم إنشاؤها تلقائياً بجانب ملفات المشروع عند أول تشغيل.

## 3. قبل التشغيل: إعدادات ضرورية

افتح ملف `config.py` وعدّل السطر التالي بوضع آيدي حسابك الحقيقي في تيليجرام
(يمكنك معرفة آيدي حسابك عبر إرسال أي رسالة لبوت `@userinfobot`):

```python
ADMIN_IDS = [123456789]  # ضع آيدي حسابك هنا بدل هذا الرقم
```

التوكن موجود بالفعل في الملف:
```python
BOT_TOKEN = "8990049234:AAGr2dcrT6nckZlFywdtBeyIVhBSjQKrnjE"
```

⚠️ **تنبيه أمني مهم:** لا تشارك هذا التوكن مع أي شخص ولا ترفع الكود إلى مستودع عام (GitHub عام مثلاً)
وهو يحتوي على التوكن، لأن أي شخص يحصل عليه يمكنه التحكم الكامل ببوتك. إن كنت قد شاركت
هذا التوكن سابقاً في مكان عام، يُفضّل توليد توكن جديد عبر @BotFather بالأمر `/revoke`.

## 4. أوامر الأدمن المتاحة

أرسل `/adminhelp` من حساب الأدمن لعرض كل الأوامر مع شرح استخدامها، وأهمها:

| الأمر | الوصف |
|---|---|
| `/addproduct الاسم\|الوصف\|السعر\|النوع\|المحتوى` | إضافة منتج جديد |
| `/delproduct <id>` | حذف منتج |
| `/editprice <id> <price>` | تعديل سعر منتج |
| `/editcontent <id>\|المحتوى الجديد` | تعديل محتوى منتج |
| `/allowrebuy <id>` | تبديل السماح بإعادة شراء منتج |
| `/listproducts` | عرض كل المنتجات مع آيديهم |
| `/addpoints <user_id> <amount>` | إضافة نقاط لمستخدم |
| `/delpoints <user_id> <amount>` | خصم نقاط من مستخدم |
| `/listusers` | عرض جميع المستخدمين |
| `/stats` | إحصائيات عامة عن البوت |

### مثال عملي لإضافة منتج

```
/addproduct بانل استضافة تجريبي|بانل بسيط لإدارة السيرفر|100|Panel|هذا هو رابط/كود البانل الحقيقي
```

## 5. مفتاح تجربة المطور

عند إرسال الكود التالي حرفياً من أي مستخدم:

```
nqGgabh35Di
```

يقوم البوت بإضافة 99999 نقطة لصاحب الرسالة تلقائياً. لتعطيل هذه الميزة كلياً،
افتح `config.py` وغيّر:

```python
DEV_SECRET_ENABLED = False
```

## 6. ملاحظات إضافية

- كل عملية شراء تُسجَّل في جدول `purchases`، ولا يمكن لنفس المستخدم شراء نفس المنتج
  مرتين إلا إذا فعّل الأدمن خاصية إعادة الشراء لذلك المنتج عبر `/allowrebuy`.
- رصيد النقاط لا يمكن أن يصبح سالباً أبداً (يتم ضبطه تلقائياً على صفر كحد أدنى).
- الكود منظم بشكل يسهل توسيعه لاحقاً (مثلاً إضافة طرق ربح نقاط جديدة في `earn_points_keyboard`
  و `earn_points_callback` داخل `handlers/user.py`).
